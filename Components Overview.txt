										Ext.js Components
***********************************************************************************************************************************************************
 An Ext JS application's UI is made up of one or many widgets called Components.

  Ext JS provides a wide range of useful Components out of the box, and any Component can easily be extended to create a customized Component.
 
  Since EXT.js is object oriented, All components are organized in proper hierarchical structure.

Key Characteristics:
................................
Base Class: 
	All Components are subclasses of the Ext.Component class. which allows them to participate in automated lifecycle management including instantiation, rendering, sizing and positioning, and destruction

Config Driven:
   Components are highly configurable through a config object passed during creation.

Lifecycle:
Components go through a lifecycle:

Instantiation
Rendering

What is xtype?
	In Ext JS, xtype is a shorthand identifier for a class of components.
	It tells Ext JS which type of component to create ‚Äî but in a more concise way than writing the full class name.						

üëâ Think of xtype as an alias for a component class.

üîë Why use xtype?
‚úÖ Shorter & cleaner configuration
‚úÖ Useful in items arrays inside containers
‚úÖ Enables lazy instantiation (Ext JS creates components only when needed)	

When you define a component with xtype, Ext JS maps it to its full class automatically.

| Class               | xtype      |
| ------------------- | ---------- |
| `Ext.button.Button` | `'button'` |
| `Ext.panel.Panel`   | `'panel'`  |
| `Ext.grid.Panel`    | `'grid'`   |
| `Ext.form.Panel`    | `'form'`   |

Use case 2: Acces TextBox Value

Ext.application({
    name: 'MyApp', //name of the application : this is namespace prefix
    launch: function () {
        Ext.create('Ext.panel.Panel', {
            title: 'Panel with TextBox and Button',
            width: 600,
            height: 500,
            bodyPadding: 10,
            items: [
                {
                    xtype: 'textfield',
                    fieldLabel: 'Name',
                    itemId: 'myTextField' // Given an itemId to find it easily 
                },
                {
                    xtype: 'button',
                    text: 'Click',
                    //listner
                    handler: function (btn) {
                        //Get the Button Object reference
                        //Get the Immediate Parent Object Reference
                        var panel = btn.up('panel')
                        //console.log(panel)
                        console.log(panel.getTitle())
                        //set the new Panel Title when i click the button
                        //  Ext.Msg.alert('')
                        panel.setTitle('Welcome To Event Listeners')
                        console.log(panel.getTitle())
                        //Get the TextField Value
                        var textfield = panel.down('#myTextField')
                        console.log(textfield.getValue())
                        Ext.Msg.alert('YourName',textfield.getValue())
                    }
                }

            ], //referes the other component
            renderTo: 'root'
        })


    }
})
üìå Key points
‚úÖ xtype: 'textfield' ‚Üí Adds the textbox.
‚úÖ itemId ‚Üí Unique ID within the container (good for .down() selector).
‚úÖ handler ‚Üí The Button‚Äôs click listener.
‚úÖ btn.up('panel') ‚Üí Gets the parent Panel.
‚úÖ panel.down('#myTextField') ‚Üí Finds the TextField inside the Panel.
‚úÖ getValue() ‚Üí Gets the current input value.

‚öôÔ∏è How it works
When the Button is clicked ‚Üí
its handler runs ‚Üí
it looks up the hierarchy to find the Panel ‚Üí
then down to find the TextField ‚Üí
then reads its value.

‚úÖ This pattern is the standard in Ext JS
up() ‚Üí navigate to a parent container
down() ‚Üí find a child by itemId, xtype, or CSS selector.

								  Event Listener Configuration
............................................................................................................................................................................................................................
üìå 1Ô∏è‚É£ handler ‚Äî What is it?
‚úÖ handler is a shortcut for the most common event (usually click on a button, or change on a field).
‚úÖ You can only define one handler per component.
‚úÖ It‚Äôs mostly used on buttons and menu items for quick click actions.

{
    xtype: 'button',
    text: 'Click Me',
    handler: function(btn) {
        Ext.Msg.alert('Hello', 'Button clicked!');
    }
}
üëâ Equivalent to:
{
    xtype: 'button',
    text: 'Click Me',
    listeners: {
        click: function(btn) {
            Ext.Msg.alert('Hello', 'Button clicked!');
        }
    }
}

üëâ So with listeners you can handle many events for the same component.


{
    xtype: 'textfield',
    fieldLabel: 'Name',
    listeners: {
        change: function(field, newValue, oldValue) {
            console.log('Value changed from', oldValue, 'to', newValue);
        },
        focus: function(field) {
            console.log('TextField focused');
        }
    }
}

üìå 4Ô∏è‚É£ When to use which?
‚úÖ Use handler

For simple, quick click actions on buttons or menu items.

Keeps your config short and clean.

‚úÖ Use listeners

For complex components (grids, panels, fields).

When you need multiple event types.

When you want to attach events declaratively in config.

 Best Practice
üëâ For simple buttons: handler
üëâ For fields, panels, grids, custom components: listeners
üëâ For multiple events: listeners is the way to go!
